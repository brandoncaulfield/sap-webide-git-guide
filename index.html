Important Concepts in SAP WebIDE

SAP WebIDE allows you to create a git repository for each project you work on and the creation of the repository is controlled from within the SAP WebIDE itself. If you want to work on an application (that you did not create yourself) from the WebIDE then you will need to clone the repository as you would any other normal git repository. Once you have made your changes you have to first pull the latest version from the remote repository into your local origin/master, merge your changes, fix any issues and then only push your changes to the remote repository if you want other developers to see your changes.


Important git commands

pull
Pull refers to when you are fetching in changes and merging them. For instance, if someone has edited the remote file you're both working on, you'll want to pull in those changes to your local copy so that it's up to date.

fetch
Fetching refers to getting the latest changes from an online repository without merging them in. Once these changes are fetched you can compare them to your local branches (the code residing on your local machine).

push
Pushing refers to sending your committed changes to a remote repository, such as a repository hosted on GitHub. For instance, if you change something locally, you'd want to then push those changes so that others may access them.


When in the SAP WebIDE you'll find most of what you need in the Git Menu near the top right of the screen. You'll also find the console towards the bottom right of the screen which can be very useful when trying to see what happened as you interact with the git repository.

A great way to see if you've successfully pushed your changes to the remote repository is to download the Git desktop client and clone the repositroy from there. Once you have the files on your desktop you can open them in your favourite IDE and see if your changes are there.

When creating a new UI5 app the web IDE will create a remote repository for you. This URL is what will be used when you set remote your newly created repository with the remote repository on GitHubs servers.

In the SAP Cloud Platform Cockpit you'll find the remote Git repository link under the versioning section after you've clicked on the HTML5 app itself.
